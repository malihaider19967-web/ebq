This prompt is designed for an AI agent to build a Keyword Intelligence & Content Strategy Layer into your system. Since you've already implemented the basic audit and the general recommendations, this focus is strictly on Keyword Coverage, Search Intent, and Competitive Gap Analysis.

AI Agent Prompt: Keyword Strategy Logic Architect
Role: Senior SEO Data Engineer and Content Strategist.

Objective: Design a rule-based logic engine to analyze "Target Keywords" against "Crawled Page Data." This engine must produce actionable "Extra Value" recommendations for the user without requiring AI during the actual audit run.

Inputs Available in the System:

Target Keyword List: A user-provided list of keywords they want to rank for.

Page Component Map: Extracted text from <title>, <meta-description>, <h1>, <h2>-<h6>, and the first 200 words of <body>.

Keyword Frequency: A count of how many times each Target Keyword appears in the full body text.

Task: Create a technical specification (pseudo-code) and a dictionary of "Recommendation Strings" for the following four logic modules:

1. The "Power Placement" Check (SEO Basics)
Logic: For the Primary Keyword (the first one in the user's list), check presence in: Title, H1, and Meta Description.

Trigger: If False for any location.

Recommendation String: Create specific alerts for each missing location (e.g., "Your primary keyword is missing from the H1, which is a high-priority signal for Google.")

2. Topical Coverage & Semantic Gap
Logic: Compare the total list of Target Keywords against the words found on the page.

Calculation: Coverage Score = (Keywords Found / Total Target Keywords) * 100.

Recommendation Logic: * If score < 50%: Suggest "Topical Expansion" and list the specific keywords that are missing.

If score > 80%: Validate as "High Topical Authority."

3. Intent-Type Alignment
Logic: Parse the Target Keywords for "Intent Triggers":

Informational: "How to", "Why", "Guide", "Tips", "Tutorial".

Utility/Transactional: "Generator", "Tool", "Maker", "Online", "Download".

Recommendation Logic: * If Informational triggers are found, recommend adding "FAQ Schema" or "Table of Contents."

If Utility triggers are found, recommend "Above-the-fold" tool placement and "Step-by-Step" instructions.

4. The "Accidental Authority" Detector
Logic: Find any phrase in the body text that has high density (>2%) but is NOT in the user's Target Keyword list and NOT in the Title/H1.

Recommendation String: "Observation: You are using the term '[Term]' frequently. If this is a relevant keyword, consider adding it to your Title or H1 to capture unexpected search traffic."

Constraints for the Agent:

Deliver the output as a JSON-like dictionary of Conditions and Response_Templates.

The logic must be strictly computational (counting, string matching, or regex) so it can be hard-coded into a Laravel/PHP or Python backend.

The tone of the recommendations should be professional, encouraging, and data-driven.

How to use this with your current system:
By adding this logic, your report for pubgnamegenerator.net would go from saying:

"Word count is 1,580" To saying:

"Word count is 1,580 (High). However, your target keyword 'PUBG font styles' only appears once. You have plenty of space to increase the coverage of this keyword to improve your ranking." This makes the report feel like a personalized manual written by a human expert.
